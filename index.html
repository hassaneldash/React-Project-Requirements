Medical Booking App

React + Laravel/Django Backend + Material UI/TailwindCSS

★ Project Overview

The application is a role-based medical appointment booking system. Patients can browse doctors and book slots, doctors can manage their availability and view schedules, and administrators have full oversight of the platform's users and configurations. You will utilize the React ecosystem libraries and best practices to deliver these requirements.

UI Framework: You must build the UI using Material UI or any other UI library except Bootstrap (Don't use Bootstrap). All routing should utilize Role-based routing using React Router.
Routes

Required Pages

Create a React app with the following pages using React Router, including handling a 404 Not Found page.

  • Doctor Search (Main page with specialty/name filters)
  • Doctor Profile & Booking (View availability and book slot)
  • Patient Dashboard (View, cancel, or reschedule appointments)
  • Doctor Dashboard (Set availability, view upcoming/past appointments)
  • Admin Dashboard (System management, approve/block users)
  • Login (Authentication for all roles)
  • Register (Patients and Doctors can register)
  • Profile Edit (Logged-in user info)
  • 404 Not Found (Catch-all for invalid URLs)

Features

Core Functional Requirements

1 Patient Capabilities API UI

  • Discovery: View list of available doctors with functional filters for specialty and name.
  • Booking: View a doctor's availability and book an appointment (date/time slot).
  • Management: View, cancel, or reschedule their appointments. Edit personal profile.
  • Validation: Ensure real-time validation of availability to completely avoid double booking.
  • Bonus Sorting: Sort doctors by rating, specialty, or availability using query parameters (e.g., sort_by=rating).
  • Bonus Pagination: Paginate between doctor list pages and admin master lists by sending a query parameter (e.g., page=2).
  • Bonus Search Debouncing: Implement a debounce utility (e.g., 500ms delay) on the search input to protect the backend API rate limits.

2 Doctor & Admin Capabilities State Routing

  • Doctor Scheduling: Set availability schedule (days/times).
  • Doctor Dashboard: View upcoming and past appointments. Manage profile (specialty, bio, contact). Optionally: mark appointment as approved or rejected, and can add notes.
  • Admin Moderation: View all users (doctors & patients). Approve or block doctors/patients.
  • Admin Config: CRUD on specialties, doctors, and other system configurations. View all appointments system-wide.

3 UI Enhancements & Auth UI Components

  • UI Components: Use Material UI or another UI library except bootstrap.
  • Authentication: Secure login/authentication handling (JWT) via Django Rest Framework + djangorestframework-simplejwt or Laravel equivalent.
  • State & Forms: Strict form validation and state management via Context API or Redux Toolkit.
  • Enhancements: Implement Dynamic Page Titles (updating the browser tab to match the view) and a Back to Top Button.
  • Notifications: Provide user feedback via a Snackbar/Toast library for successful bookings or errors.
  • Empty States & Skeletons: Display Skeleton components while fetching doctor availability. Show a clear "Empty State" graphic if no appointments exist.

4 Architecture & Best Practices Architecture

  • Role-Based Routing: Implement role-based routing using React Router to secure Patient, Doctor, and Admin views.
  • API Error Handling: Implement global or component-level error handling. Display user-friendly error messages (via Toasts) if fetching fails.
  • Dark Mode Toggle: Implement a functional dark mode switch for the medical platform.
  • Bonus Custom Hooks: Create custom hooks or utility files (e.g., Axios instances) to separate backend API requests from UI components.
  • Bonus i18n: Language dropdown (e.g., English, Arabic) changing the text and switching layout to RTL for Arabic.
  • Bonus Lazy Loading: Use React.lazy() and <Suspense> to code-split page routes, ensuring the Admin Dashboard code is only loaded for Admins.
